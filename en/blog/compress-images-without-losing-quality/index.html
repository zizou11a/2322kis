What Is Image Compression? A Plain-English Explanation

Image compression is the process of reducing the file size of an image by encoding its data more efficiently. Every image file contains millions of pixels, each storing colour information. Compression algorithms look for patterns and redundancies in that data and represent them more compactly — similar to how a zip file shrinks a folder without deleting anything from it.

The result is a smaller file that takes less storage space, loads faster on websites, and transfers more quickly over email or messaging apps — often with no visible difference to the human eye.

Lossy vs Lossless Compression: Which Should You Use?

There are two fundamental approaches to image compression. Understanding the difference tells you exactly which one to use for any image.

Property Lossless Lossy
Quality loss None — bit-perfect Imperceptible at 75–85%
File size reduction 20–40% 60–80%
Best for Logos, icons, screenshots, text-heavy images Photographs, hero images, product photos
Formats PNG, WebP (lossless mode), GIF JPG, WebP (lossy mode), AVIF
Can re-edit? Yes — no generation loss Each re-save degrades quality

The Simple Decision Rule

  • Does the image contain text, fine lines, a logo, or a diagram? → Use lossless (PNG or lossless WebP).
  • Is it a photograph — people, landscapes, products? → Use lossy (JPG at 80%, or lossy WebP).
  • Is it going on a website and file size is critical? → Convert to WebP regardless of content type.
🎓 Expert Tip — IMGVO

Never re-compress an already-compressed JPG. Each time a JPG is saved, the lossy codec runs again and degrades quality further. Always keep one lossless master copy (PNG or TIFF) and generate your compressed JPG or WebP from that original — not from a previously compressed version.

How to Compress by Format: JPG, PNG, and WebP

Compressing JPG Images

JPG is a lossy format by nature — it was designed for photographs and achieves impressive compression by discarding fine colour details the eye doesn't easily perceive. The key variable is the quality setting, usually expressed as a number from 1 to 100.

  • Quality 85–95: Near-original quality, modest file reduction. Best for print-ready images.
  • Quality 75–85: The sweet spot for web. 60–70% smaller than original with no visible difference on screen.
  • Quality 60–75: Aggressive compression, useful for thumbnails or low-bandwidth contexts. Some softness may appear on close inspection.
  • Below 60: Visible blocking and colour banding artifacts — avoid for anything public-facing.

Compressing PNG Images

PNG is a lossless format, which means standard PNG compression cannot degrade quality — it can only reorganise the data more efficiently. However, you can achieve significant reductions using two additional techniques:

  • PNG quantisation (pngquant) — reduces the colour palette from 16 million colours to 256 without visible loss on most images. Typically achieves 50–70% size reduction.
  • Converting to WebP — for photographs stored as PNG, converting to lossy WebP can reduce file size by 70–90% with no visible quality difference.

⚠️ Important: If your PNG contains transparency and you need to keep it, do not convert to JPG (which doesn't support transparency). Use PNG lossless compression, lossless WebP, or AVIF instead.

Compressing WebP Images

WebP supports both lossy and lossless modes in the same format. A WebP at quality 80 is typically 25–35% smaller than an equivalent JPG at quality 80, with the same visual result. If you already have WebP files that are too large, re-compress them in lossy mode at quality 75–80.

Image Compression Best Practices for Websites

Compressing individual images is a good start. For websites, there are a few additional practices that make a significant difference to performance and SEO.

1. Serve images in modern formats

Use WebP as your primary format with JPG as a fallback for older browsers. You can do this with the HTML <picture> element:

💡 HTML pattern: Use <source type="image/webp" srcset="image.webp"> inside a <picture> tag, with <img src="image.jpg"> as the fallback. All modern browsers will use WebP; only very old ones fall back to JPG.

2. Resize before you compress

The single biggest mistake is uploading a 4000×3000 px photo for a 400×300 px thumbnail. No amount of compression compensates for unnecessary pixels. Always resize the image to the exact display dimensions first, then compress.

3. Set a target file size by context

  • Hero images: 100–200 KB maximum
  • Inline article images: 60–120 KB maximum
  • Thumbnails: 10–30 KB maximum
  • Icons / logos: Under 10 KB (use SVG where possible)

4. Use lazy loading

Add loading="lazy" to all images below the fold. This defers their download until the user scrolls near them, reducing initial page load time without any quality impact.

5. Write descriptive alt text

Alt text doesn't affect file size, but it directly affects image SEO. Every <img> tag should have a meaningful alt attribute describing the image content, including relevant keywords where natural.

Frequently Asked Questions About Image Compression

Can you compress an image without losing quality?
Yes — using lossless compression, you can reduce image file sizes by 20–40% with absolutely no quality loss. For larger reductions (up to 80%), smart lossy compression removes only data that is imperceptible to the human eye, so the result looks identical even though the file is much smaller.
What is the best format for compressing images?
WebP is the best format for web images — it is 25–35% smaller than JPG and 26% smaller than PNG at the same quality. AVIF is even more efficient but has less browser support. For maximum compatibility, use JPG for photos and PNG for graphics with transparency, and compress both before publishing.

What is the difference between lossy and lossless compression?
Lossless compression reduces file size without discarding any image data — the decompressed file is bit-for-bit identical to the original. Lossy compression achieves much larger reductions by permanently removing subtle details the human eye can't easily detect. JPG is inherently lossy; PNG uses lossless compression; WebP supports both modes.
Does compressing an image reduce its resolution?
No. Compression reduces file size by encoding image data more efficiently or removing imperceptible details — it does not change the pixel dimensions or resolution. A 4000×3000 px image stays 4000×3000 px after compression. If you want to reduce resolution, that is a separate step called resizing.
What quality level should I use when compressing JPG?
For web use, a JPG quality setting between 75 and 85 out of 100 gives the best balance of file size and visual quality. Below 70 introduces visible artifacts; above 90 provides diminishing returns with disproportionately larger file sizes. For print, use 90–95.
How much can I reduce an image file size without visible quality loss?
With smart lossy compression, most JPG photos can be reduced by 60–80% with no visible difference on screen. PNG files with large flat-colour areas can be reduced even more — up to 90% — using tools like pngquant or converting to WebP. Photographic PNGs typically compress 40–60%.
